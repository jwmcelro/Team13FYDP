Tables

Design tables according to best practices to ensure that they are readable for students using screen readers.

  • Microsoft PowerPoint
  • Microsoft Word
  • Google Docs & Slides

Microsoft PowerPoint

This workflow includes the following task: best practices for creating tables, header rows and column headings, table structure don'ts, using alt text, using greyscale.

1. Table Structure Best Practices

Following best practice advice for table structure helps increase the clarity and readability of the table.

Step 1: Only use one row in the header.

Step 2: Make sure to include a caption or title for your table.

2. Use a Header Row and Column Headings

Using proper headers ensures that screen readers can properly convey the information presented in the table to students.

Step 1: Open the "Table Design" tab from navigation bar and select the "Header Row" checkbox. This will ensure screen readers know that the first row in your table is the headings.


Step 2: Place your cursor in the first cell on the top row of your table to add column headings. Type the name of the selected column and press the tab key to move to the next one.

3. Table Structure Don'ts

Avoiding improper uses of tables can help increase clarity and readability for students.

Step 1: Do not use tables as a means of laying out a page. Restrict the use of tables to all but the most basic tabular data.


Step 2: Never merge or split cells.


Step 3: Try to not leave any cells, rows, or columns blank.

4. Adding Alt Text to Tables

Ensures that tables contain text descriptions that can be read by a screen reader to help visually impaired students.

Step 1: Right click on the table and select "Edit Alt Text".



Step 2: Type 1-2 sentences to describe the table.

5. Grayscale Table Colour

Choose a table structure that uses minimal colour to accomodate for colour blindness and to reduce distraction.

Step 1: Select your Table, click the Table Design tab, and choose a table design with limited colours.
